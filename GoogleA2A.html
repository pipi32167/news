A2A协议简介

谷歌的A2A（Agent-to-Agent）协议是2025年推出的开放标准，旨在实现AI智能体间的无缝通信与协作。该协议支持多模态数据处理（文本、音频、视频），基于HTTP、SSE、JSON-RPC等标准，易于集成，已被广泛应用于金融、医疗等领域。

核心功能

支持任务分解、信息同步和多代理协作，打破系统孤岛。

技术基础

基于HTTP等成熟标准，确保稳定性和可扩展性。

应用场景

覆盖招聘、电商物流、科研协作等复杂任务。

MCP与A2A协议对比

谷歌的模型上下文协议(MCP)和代理对代理协议(A2A)针对AI生态系统中的不同交互需求，互为补充。理解它们的差异有助于开发者选择适合的协议实现AI系统。

MCP（模型上下文协议）

用于AI代理与工具、API或外部服务的交互，适合需要直接调用外部资源或处理上下文数据的场景。

适用场景：

  • 调用外部API（如天气服务、数据库）
  • 工具集成（搜索引擎、文件系统）
  • 上下文管理（保存用户对话历史）
  • 单一代理执行简单任务

特点：

  • 侧重于"代理-工具"交互
  • 强调数据格式和上下文的标准化
  • 简化API调用流程

A2A（代理对代理协议）

用于多个AI代理之间的通信与协作，适合复杂任务分解、代理间信息共享或协同工作场景。

适用场景：

  • 多代理协作（如旅行规划、医疗诊断）
  • 任务分解（将复杂任务分配给专门代理）
  • 跨平台互操作（不同供应商代理协作）
  • 复杂、多步骤任务处理

特点：

  • 侧重于"代理-代理"交互
  • 支持动态任务分配和多模态数据
  • 实现代理间标准化通信
特性 MCP A2A
交互对象 代理与工具/API 代理与代理
数据类型 上下文、API响应 多模态（文本、音频、视频）
复杂性 单一任务 复杂任务分解
标准 HTTP、JSON HTTP、SSE、JSON-RPC
生态支持 工具集成 跨平台代理协作

实际案例代码对比

使用MCP调用天气API

简单场景：用户询问"巴黎明天天气如何"

from mcp import MCPClient

# 创建MCP客户端连接API服务
client = MCPClient(api_key="YOUR_API_KEY")

# 调用天气API获取数据
weather = client.call_api(
    "openweathermap", 
    {"location": "Paris", "date": "tomorrow"}
)

# 处理并展示结果
print(f"巴黎明天天气: {weather['temp']}°C, {weather['condition']}")

使用A2A协调多代理协作

复杂场景：用户要求"计划巴黎5天行程"

from a2a import A2AClient

# 连接各个专业代理
weather_client = A2AClient("http://localhost:8001")
search_client = A2AClient("http://localhost:8002")
planner_client = A2AClient("http://localhost:8003")

# 协调多代理协作
async def plan_paris_trip():
    # 1. 获取巴黎天气预报
    weather = await weather_client.call_method(
        "get_forecast", {"location": "Paris", "days": 5}
    )
    
    # 2. 查询巴黎热门景点
    attractions = await search_client.call_method(
        "search", {"query": "Paris top attractions"}
    )
    
    # 3. 根据天气和景点生成行程计划
    plan = await planner_client.call_method(
        "generate_plan", 
        {"weather": weather, "attractions": attractions}
    )
    
    return plan

协议选择指南

在实际应用中，MCP和A2A常结合使用，MCP负责工具层交互，A2A负责代理层协作。选择时需根据以下因素判断：

选择MCP的场景

当任务仅需单一代理调用API或工具完成，无需多代理协作时。

选择A2A的场景

当任务复杂，需要多个专业化代理协同工作，或跨平台代理交互时。

同时使用两者的场景

在构建完整生态系统时，A2A处理代理间协作，而各代理内部使用MCP调用工具和API。

代码示例

以下是使用A2A协议实现的天气代理示例，展示多代理协作能力。

天气代理（WeatherAgent.py）

from a2a import A2AServer, AgentCard
import requests

class WeatherAgent:
    def get_weather(self, location):
        api_key = "YOUR_OPENWEATHERMAP_API_KEY"
        url = f"http://api.openweathermap.org/data/2.5/weather?q={location}&appid={api_key}"
        response = requests.get(url)
        data = response.json()
        if data.get("cod") == 200:
            return f"{location}: {data['main']['temp']}°C, {data['weather'][0]['description']}"
        else:
            return "Weather data not found."

if __name__ == "__main__":
    agent_card = AgentCard(
        name="WeatherAgent",
        description="Provides weather information for a given location.",
        skills=["get_weather(location)"]
    )
    server = A2AServer(agent_card, WeatherAgent())
    server.run(port=8001)

A2A协议应用场景

A2A协议正在多个行业得到应用，以下是几个具有代表性的案例：

医疗协作

多个医疗AI代理协同工作，一个分析医学图像，一个处理病历数据，第三个整合诊断建议，为医生提供全面信息。

智能制造

工厂中的AI代理监控设备状态、预测维护需求并协调生产调度，实现全自动化生产流程。

增强搜索

搜索引擎与专业领域AI代理协作，为复杂查询提供精确答案，而非仅返回相关网页列表。
